# To Epics

Break target architecture work into user-confirmed epics, run a `grill-me` clarification session, validate terminology against `GLOSSARY.md`, and list those epics in Phase E of the target architecture document.

Use this skill only after the user selects the target architecture to link to and provides an epic name, phase, description, and main capability, or explicitly confirms a suggested epic name, phase, description, and main capability.

Before creating an epic, update the private lab root `GLOSSARY.md` with `ubiquitous-language` when important terminology is missing or ambiguous.

Before writing the epic file, run `grill-me` to clarify scope, requirements, dependencies, visual design, solution design, SRE concerns, and open questions.
