# Toolkit Boundary

This file defines what belongs to the active AI Architecture Toolkit.

The repository may contain public supporting material that is useful for documentation, examples, experiments, or project maintenance. Those files can stay public without being treated as active toolkit content.

## Active Toolkit Content

The active toolkit consists of:

- `agents/*.md` - Agent role definitions.
- `skills/*/SKILL.md` - Skill entrypoints.
- `skills/*/README.md` - Skill documentation.
- `skills/*/templates/` - Templates owned by a skill.
- `skills/*/scripts/` - Helper scripts owned by a skill.
- `principles/*.md` - Architecture and AI usage principles.
- `templates/*.md` - Shared architecture templates not owned by a skill.
- `governance/*.md` - Public governance guidance.

## Generated Or Context-Specific Content

The following content is public but not part of the active toolkit by default:

- `capabilities/` - Generated capability overview documents.
- `GLOSSARY.md` - Working glossary generated by the ubiquitous-language skill.
- `CONTEXT.md` - Working project context.
- `www/` - Website or presentation layer files.
- `examples/` - Reference scenarios and examples.
- `BACKLOG.md` - Project backlog.
- `CHANGELOG.md` - Project change history.

## Ignore Rules

Toolkit scanners and automation should read `.toolkitignore` before collecting active toolkit content.

`.toolkitignore` does not affect git tracking. It only defines what toolkit-aware scripts and agents should skip when building an active toolkit view.

## Public Repository Rule

Do not add company-confidential information, customer names, internal system names, proprietary architecture details, credentials, or non-public business context to this repository. Use a separate private company lab repository for real-world testing.
