# Contribution Guide

## Contribution Standard

Contributions should be practical, reviewable, and aligned with the toolkit structure.

## Guidelines

- Use clear markdown with concise headings.
- Prefer reusable templates, checklists, prompts, and examples.
- State assumptions and intended use.
- Keep examples generic unless a domain-specific scenario is intentional.
- Avoid embedding confidential, customer-specific, or proprietary information.

## Review

Contributions should be checked against architecture principles, AI agent principles, and quality guardrails before merging.
