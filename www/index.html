Use purpose-built agents for architecture work.

Start each engagement with a role brief that sets scope, responsibilities, and expected outputs for a specific architecture discipline.

Solution Architect Agent

Creates high-level solution designs from capability overviews, records material ADRs, and embeds rendered Mermaid diagrams.

Enterprise Architect Agent

Owns governance, principles, target architecture, roadmap alignment, and enterprise-impacting architecture decisions.

Business Analyst Agent

Clarifies business needs, scope, stakeholders, outcomes, terminology, and traceability for architecture work.

Data Architect Agent

Guides canonical data objects, ownership, lifecycle, privacy, quality, integration, and governance decisions.

Integration Architect Agent

Shapes APIs, events, data flows, reliability, observability, and integration ownership.

API Governance Agent

Reviews API contracts for naming, versioning, security, lifecycle risk, and developer experience.

Architecture Review Board Agent

Prepares review findings, tracks decisions, and checks alignment with principles and guardrails.

Security Architect Agent

Identifies threats, controls, privacy concerns, compliance constraints, and operational security requirements.

Lead Developer Agent

Turns approved architecture direction into buildable, testable, operable delivery plans.

Turn architecture habits into repeatable skills.

Skills encode the workflow, templates, scripts, and guardrails needed to produce consistent artifacts.

Target architecture document (ADD)

Connect capabilities to strategy, business architecture, data, applications, integration, technology, gaps, and roadmap themes.

High-level solution design

Produce implementation-oriented solution designs with chapter guidance, Mermaid sources, rendered SVGs, and linked ADRs.

Create Draw.io diagram

Create editable Draw.io architecture diagrams from templates for capability context, components, data models, and integrations.

Capability overview

Write business capability overviews under capabilities/ from a standard template.

Architecture decision record (ADR)

Capture durable architecture decisions as concise ADRs when choices are hard to reverse and based on real trade-offs.

Ubiquitous language

Extract a shared glossary, flag ambiguous terms, and define canonical applications, data objects, and relationships.

Grill me

Challenge plans, assumptions, constraints, terminology, risks, and decisions until the design context is explicit.

Write a principle

Draft architecture principle documents with identifiers, rationale, consequences, and reviewable guidance.

Create an agent

Create role-specific architecture agent definitions under agents/ from the repository template.

Create a skill

Create reusable skills with progressive disclosure, templates, scripts, and clear trigger guidance.

Keep public toolkit content separate from private company work.

Use this repository as the public source of reusable architecture assets. Add it to a private company lab as a submodule when testing against real context.

What the toolkit creates.

The output is plain markdown, easy to review, version, and improve with your existing architecture process.

Capability and architecture folders

Capability overviews and solution architecture documents under the matching repository folders.

Decision records

Numbered ADRs linked from the technical design choices they support.

Glossaries and principles

Shared language in GLOSSARY.md plus reusable principle documents under principles/.

Rendered diagrams

Mermaid .mmd sources and Draw.io .drawio files paired with same-basename .svg exports when embedded in design documents.
