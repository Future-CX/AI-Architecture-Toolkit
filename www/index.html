Use purpose-built agents for architecture work.

Start each engagement with a role brief that sets scope, responsibilities, and expected outputs for a specific architecture discipline.

Solution Architect Agent

Creates solution architecture designs from capability overviews, records material ADRs, and embeds rendered Draw.io diagrams.

Enterprise Architect Agent

Owns governance, principles, target architecture, roadmap alignment, and enterprise-impacting architecture decisions.

Business Analyst Agent

Clarifies business needs, scope, stakeholders, outcomes, terminology, and traceability for architecture work.

Data Architect Agent

Guides canonical data objects, ownership, lifecycle, privacy, quality, integration, and governance decisions.

Integration Architect Agent

Owns integration principles and patterns for APIs, events, data flows, reliability, observability, and integration ownership.

Lead Developer Agent

Turns approved solution and integration designs into buildable, testable, operable delivery plans.

Turn architecture habits into repeatable skills.

Skills encode the workflow, templates, scripts, and guardrails needed to produce consistent artifacts.

Target architecture document (ADD)

Connect capabilities to strategy, business architecture, data, applications, integration, technology, and epics to build.

Solution architecture design

Produce implementation-oriented solution designs with chapter guidance, editable Draw.io sources, rendered SVGs, and linked ADRs.

Data architecture design

Create data-object-specific designs with data flow diagrams, integration traceability, ownership, lifecycle, and Phase C linkage.

Integration design

Document APIs, events, files, batches, messaging, and orchestration with contracts, failure handling, and operations.

Problem to solution

Frame a problem, compare solution options with pros and cons, and recommend a path for senior stakeholder decisions.

Create Draw.io diagram

Create editable Draw.io architecture diagrams from templates for capability context, components, data models, and integration design maps.

To epics

Break target architecture work into user-confirmed epics and list each epic in Phase E of the target architecture document.

Capability overview

Write business capability overviews under capabilities/ from a standard template.

Architecture decision record (ADR)

Capture durable architecture decisions as concise ADRs when choices are hard to reverse and based on real trade-offs.

Ubiquitous language

Extract a shared glossary, flag ambiguous terms, and define canonical applications, data objects, and relationships.

Grill me

Challenge plans, assumptions, constraints, terminology, risks, and decisions until the design context is explicit.

Write a principle

Draft architecture principle documents with identifiers, rationale, consequences, and reviewable guidance.

Publish to Confluence

Publish markdown or HTML toolkit content to Confluence pages through the Confluence REST API.

Keep public toolkit content separate from private company work.

Use this repository as the public source of reusable architecture assets. Add it to a private company lab as a submodule when testing against real context.

What the toolkit creates.

The output is plain markdown, easy to review, version, and improve with your existing architecture process.

Capability overviews

Business capability documents under capabilities/ with scope, outcomes, owners, and related systems.

Solution architectures

Target architecture and solution architecture design documents under solution-architectures/.

Data architecture designs

Data-object-specific designs under data-architectures/, linked from Phase C of the target architecture document.

Epics

User-confirmed epic files under requirements/, listed in Phase E with traceability to target architecture.

Problem-to-solution briefs

Stakeholder-ready decision documents under problem-to-solutions/ with options, trade-offs, and recommendations.

Decision records

Numbered ADRs linked from the technical design choices they support.

Glossary

Shared domain language in GLOSSARY.md, including canonical terms, applications, data objects, and relationships.

Principles

Reusable architecture principle documents under principles/.

Rendered diagrams

Editable Draw.io .drawio sources paired with same-basename .svg exports when embedded in design documents.
